## 一、项目背景

### 业务场景
社区超市提供两类核心服务：
1. **传统零售**：食品类商品正常销售（新鲜果蔬、米面粮油、酒水饮料、零食、半成品、鲜食等）
2. **工具租赁**：提供可出租的工具供社区居民使用

### 解决的痛点
- **购买成本高**：很多设备和工具使用频率低，购买后闲置造成浪费
- **租赁成本高**：传统出租方距离顾客远，配送成本高
- **品类不齐全**：市面上没有全品类工具出租方

### 解决方案
社区超市距离居民足够近，顾客可以自己借还工具，降低租赁成本，实现资源共享。

## 三、工具租赁业务流程

### 用户端流程
```
浏览工具 → 选择租用日期和数量 → 支付押金 → 到店取工具
    ↓
使用完毕 → 出示二维码 → 操作员扫码确认归还 → 退还押金
```

### 操作员流程
```
扫描用户订单二维码 → 确认租用/归还 → 系统计算租金 → 退还押金
```

### 订单状态流转
| 状态      | 说明           |
| ------- | ------------ |
| 待确认 (0) | 用户下单，等待操作员确认 |
| 已租用 (1) | 操作员确认，用户开始使用 |
| 已归还 (2) | 用户归还，押金结算完成  |
| 已取消 (3) | 用户取消订单       |

### 租金计算规则
```
租用天数 = 当前日期 - 确认租用时间
计费天数 = max(0, 租用天数 - 免租天数)
实际租金 = 日租金 × 计费天数 × 数量
退还押金 = 押金 - 实际租金
```

## 四、工具分类

### 主要分类
| 分类   | 示例工具              |
| ---- | ----------------- |
| 全屋清洁 | 擦窗机、除螨仪、扫地机器人、吸尘器 |

- 微型二维码
- 暗码/隐形码
- 线激光3D轮廓扫描仪
- 三维扫描仪

## 七、租赁提醒通知功能

### 通知类型
| 类型 | 触发时机 | 说明 |
|------|----------|------|
| 租用开始 | 操作员确认租用 | 通知用户租用成功 |
| 即将到期 | 归还日期前1天 | 提醒用户即将到期 |
| 已到期 | 超过归还日期 | 提醒用户尽快归还 |
| 归还确认 | 用户归还成功 | 通知押金退还金额 |

### 技术实现
- 微信小程序订阅消息
- 定时任务每小时检查并发送通知
- 通知记录表存储发送记录

### 配置要求
需要在 `application.yml` 配置微信订阅消息模板ID：
```yaml
wechat:
  subscribe:
    template:
      rental-start: 模板ID
      expiring-soon: 模板ID
      expired: 模板ID
      return-confirmed: 模板ID
```
